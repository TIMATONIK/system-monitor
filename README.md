# 🖥️ Linux System Monitor

![Python](https://img.shields.io/badge/Python-3.10%2B-blue?logo=python&logoColor=white)
![Linux](https://img.shields.io/badge/Platform-Linux-orange?logo=linux&logoColor=white)
![License](https://img.shields.io/badge/License-MIT-green)
![Telegram](https://img.shields.io/badge/Telegram-Bot-blue?logo=telegram&logoColor=white)

Инструмент мониторинга Linux системы в реальном времени с живым Rich-дашбордом в терминале и полноценным Telegram-ботом. Собирает метрики каждые 30 секунд, сохраняет историю в CSV и отправляет автоматические алерты при превышении порогов.

## 📊 Собираемые метрики

| Категория | Метрики |
|-----------|---------|
| **CPU** | Общая загрузка (%), загрузка по каждому ядру, тактовая частота (МГц), количество ядер |
| **Память** | ОЗУ: использовано / всего (ГБ), процент заполнения; Своп: использовано / всего (ГБ), процент |
| **Диски** | Для каждого раздела: устройство, точка монтирования, тип ФС, использовано / всего / свободно (ГБ), % заполнения |
| **Температура** | Все доступные сенсоры через `psutil` (coretemp, k10temp, nvml и др.), включая GPU |
| **Сеть** | Суммарно отправлено/получено (МБ/ГБ), текущая скорость (КБ/с), статистика по каждому интерфейсу |

## 🤖 Команды Telegram бота

| Команда | Описание |
|---------|----------|
| `/start` | Приветствие и список всех доступных команд |
| `/status` | Полный дашборд всех метрик прямо сейчас |
| `/cpu` | Детальная информация по CPU: нагрузка по ядрам, частота |
| `/ram` | Детальная информация по оперативной памяти и свопу |
| `/disk` | Все диски и разделы с процентом заполнения |
| `/temp` | Температуры всех доступных сенсоров |
| `/net` | Сетевой трафик, скорости, список интерфейсов с адресами |
| `/history` | ASCII-графики изменения CPU, RAM и сети за последний час |
| `/alerts` | Текущие настройки порогов автоматических алертов |
| `/setalert [метрика] [порог]` | Изменить порог алерта на лету |

### Настройка алертов через `/setalert`

```
/setalert cpu 85        — алерт при CPU > 85%
/setalert ram 80        — алерт при RAM > 80%
/setalert disk 90       — алерт при заполнении диска > 90%
/setalert temp_cpu 80   — алерт при температуре CPU > 80°C
```

**Доступные ключи метрик:** `cpu`, `ram`, `disk`, `temp_cpu`

**Пороги по умолчанию:**

| Метрика | Порог |
|---------|-------|
| CPU | 90% |
| RAM | 90% |
| Диск | 95% |
| Температура CPU | 85°C |

> Повторный алерт по той же метрике отправляется не чаще одного раза в 5 минут.

## 🔔 Автоматические алерты

Бот автоматически отправляет уведомление в Telegram когда:

- 🔥 **CPU** превышает порог (по умолчанию 90%)
- 🧠 **RAM** превышает порог (по умолчанию 90%)
- 💾 **Любой диск** превышает порог заполнения (по умолчанию 95%)
- 🌡️ **Температура CPU** превышает порог (по умолчанию 85°C)

Алерты начинают отправляться после первой команды `/start` в боте.

## 🚀 Установка

### 1. Клонировать репозиторий

```bash
git clone https://github.com/TIMATONIK/system-monitor.git
cd system-monitor
```

### 2. Создать виртуальное окружение (рекомендуется)

```bash
python -m venv .venv
source .venv/bin/activate
```

### 3. Установить зависимости

```bash
pip install -r requirements.txt
```

### 4. Установить lm-sensors (для температур)

```bash
# Arch Linux / CachyOS
sudo pacman -S lm_sensors
sudo sensors-detect --auto

# Ubuntu / Debian
sudo apt install lm-sensors
sudo sensors-detect --auto
```

### 5. Получить токен Telegram бота

1. Написать [@BotFather](https://t.me/BotFather) в Telegram
2. Выполнить команду `/newbot`
3. Следовать инструкциям и скопировать полученный токен

## ▶️ Запуск

```bash
export TELEGRAM_TOKEN='ваш_токен_от_BotFather'
python monitor.py
```

Или в одну строку:

```bash
TELEGRAM_TOKEN='ваш_токен' python monitor.py
```

После запуска:
- В терминале откроется живой Rich-дашборд (обновляется каждые 5 секунд)
- Метрики начнут собираться каждые 30 секунд
- История сохраняется в файл `metrics_history.csv`
- Напишите `/start` боту в Telegram — он зарегистрирует ваш чат и начнёт слать алерты

Остановка: `Ctrl+C`
